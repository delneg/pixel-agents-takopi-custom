## Step 2: Agent manager using takopi's ClaudeRunner — DONE

Implemented in `server/agent_manager.py`:
- Uses `ClaudeRunner(dangerously_skip_permissions=True)` from `takopi.runners.claude`
- `set_run_base_dir(self._cwd)` / `reset_run_base_dir(token)` for CWD
- Iterates `runner.run(prompt, session.resume_token)` async iterator

- Error handling with fallback `agentToolsClear` + `agentStatus: 'waiting'`

## Step 3: Event mapper — TakopiEvent → pixel-agents messages — DONE

Implemented in `server/event_mapper.py`:
- `StartedEvent` → `agentStatus: 'active'`
- `ActionEvent(phase="started")` → `agentToolStart` or `subagentToolStart` (if `parent_tool_use_id` present)
- `ActionEvent(phase="started", kind="subagent")` → prepends "Subtask:" to status
- `ActionEvent(phase="completed")` → `agentToolDone` / `subagentToolDone` / `subagentClear`
- `CompletedEvent` → `agentToolsClear` + `agentStatus: 'waiting'` + `agentAnswer` (with answer text)
- `needs_permission_timer()` / `cancels_permission_timer()` helper functions
- Tool name for animation lookup comes from `event.action.detail["name"]` — maps to existing `STATUS_TO_TOOL` in webview

**Not yet verified:** Sub-agent event flow hasn't been tested end-to-end with a real Task tool invocation.

## Step 7: Permission timer (Python async) — DONE

Implemented in `server/timer_manager.py`:
- `TimerManager` class with `start_permission_timer(agent_id, tg, session)` — launches a 7s sleep in the task group
- On timeout, checks both parent-level and sub-agent tools for non-exempt ones
- Emits `agentToolPermission` for the parent + `subagentToolPermission` for each stuck sub-agent Task
- `cancel_permission_timer(agent_id)` — cancels via `anyio.CancelScope`

Sub-agent tracking in `AgentSession`:
- `active_tool_names: dict[str, str]` — parent tool_id → tool_name
- `active_subagent_tool_names: dict[str, dict[str, str]]` — parentToolId → {subToolId → toolName}
- `clear_activity()` method resets all tracking state

`dangerously_skip_permissions` now configurable via `PIXEL_AGENTS_SKIP_PERMISSIONS` env var (defaults to `"1"` = skip, set to `"0"` to require permissions).

**Limitation:** takopi does not currently translate `bash_progress` / `mcp_progress` events (`StreamEventMessage` is dropped silently), so the permission timer cannot be restarted when a long-running Bash command produces output. This differs from the extension which restarts the timer on bash/mcp progress.

## Step 8: Layout persistence — DONE

Implemented in `server/layout_store.py`:
- `read_layout()` — reads `~/.pixel-agents/layout.json`
- `write_layout()` — atomic write via `.tmp` + `os.replace()`, marks own-write timestamp
- `ensure_layout()` — file → bundled default fallback
- `load_default_layout()` — reads `assets/default-layout.json`
- `watch_layout_file(broadcast)` — watches layout file with `watchfiles.awatch()`, skips own-write changes (1s grace), broadcasts `layoutLoaded` on external changes

Server handles `saveLayout`, `exportLayout`, `importLayout` WebSocket messages. Layout watcher runs as background task in the lifespan task group.

## How to Run

```sh
# 1. Build webview
cd webview-ui && npm install && npm run build && cd ..

# 2. Install server (needs Python 3.14 for takopi)
uv venv --python 3.14 && uv pip install -e server

# 3. Run (from project root)
uv run uvicorn server.app:app --host localhost --port 8765

# 4. Open http://localhost:8765
```

## Known Issues / Open Items

1. **No furniture assets** — This repo lacks `floors.png` and `furniture/` directory. Generated by the asset pipeline scripts. Office renders walls and floor colors but no furniture.
2. **bash/mcp progress events** — takopi drops `StreamEventMessage` silently, so the permission timer can't restart on long-running Bash output (unlike the extension).
3. **CWD** — Server uses the directory you run `uvicorn` from as Claude's working directory.
4. **Prompt sending not tested** — Requires `claude` CLI available on PATH. The WebSocket plumbing and ClaudeRunner integration are in place but untested end-to-end with a real Claude process.
5. **`PIXEL_AGENTS_SKIP_PERMISSIONS`** — Defaults to `"1"` (skip permissions). Set to `"0"` to enable permission prompts.
